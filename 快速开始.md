# Computer Use Agent - 快速开始

## 🚀 一键启动

```bash
cd ai-app-lab/demohouse/computer_use
./start-all.sh
```

## 🌐 访问应用

打开浏览器访问: **http://localhost:3000**

## 🎮 使用步骤

1. **等待服务启动** (约 10-15 秒)
2. **打开浏览器** 访问 http://localhost:3000
3. **查看沙箱** 页面会自动加载本地沙箱 `local-sandbox`
4. **输入指令** 在输入框中输入自然语言指令,例如:
   - "打开浏览器并访问百度"
   - "截取当前屏幕"
   - "创建一个新的文本文件"
5. **查看执行** AI 代理会自动分析并执行任务

## 🛑 停止服务

```bash
./stop-all.sh
```

## 📊 服务状态

启动后的服务列表:

| 服务 | 地址 | 说明 |
|------|------|------|
| Frontend | http://localhost:3000 | Web 界面 |
| Planner | http://localhost:8089 | Agent 规划器 |
| MCP Server | http://localhost:8000 | MCP 协议服务 |
| Tool Server | http://localhost:8102 | 工具执行服务 |

## 🔍 查看日志

```bash
# 查看所有日志
tail -f /tmp/frontend.log
tail -f /tmp/planner.log
tail -f /tmp/mcp_server.log
tail -f /tmp/tool_server.log

# 或者同时查看所有日志
tail -f /tmp/*.log
```

## ⚙️ 配置

API Key 已配置在 `planner/config.toml`:
```toml
api_key = "YOUR_API_KEY_HERE"
```

## 🆘 常见问题

### 端口被占用
```bash
# 检查端口占用
lsof -i :3000
lsof -i :8089
lsof -i :8000
lsof -i :8102

# 停止所有服务后重新启动
./stop-all.sh
./start-all.sh
```

### 服务启动失败
```bash
# 查看对应服务的日志
tail -50 /tmp/planner.log
tail -50 /tmp/mcp_server.log
tail -50 /tmp/tool_server.log
tail -50 /tmp/frontend.log
```

### 前端无法访问
```bash
# 等待 10-15 秒让服务完全启动
sleep 15

# 检查前端是否启动
curl http://localhost:3000
```
